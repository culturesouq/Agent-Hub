Relevance — LLM prompt: "Does this content relate to this mandate: [mandate]? Score 0-100." Fail if score < 50.
Durability — LLM prompt: "Is this fact stable over time, or is it ephemeral (price, today's news, live event)? Answer: stable or ephemeral." Fail if ephemeral.
Privacy — LLM prompt: "Does this content contain personal identifiable information (names, emails, phone numbers, addresses)? Answer: yes or no." Fail if yes.
No duplication — embed content, vector search existing operator_kb for this operator. If any result has cosine similarity > 0.92 → skip silently.
No contradiction — same vector search. If similarity > 0.85 with a verified entry → store as pending with flagReason: 'Possible contradiction with existing knowledge'. Don't fail — just flag.
Confidence score — compute from relevance score (0-100 → 0-100). If < 60 → skip. If ≥ 85 → verificationStatus: 'verified'. Otherwise → verificationStatus: 'pending'.
Source trust — if sourceUrl contains .gov, .edu, [reuters.com](https://reuters.com), [bbc.com](https://bbc.com), [nature.com](https://nature.com), who.int → sourceTrustLevel: 'external_verified'. Otherwise → sourceTrustLevel: 'external_unverified'.
