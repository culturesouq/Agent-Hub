OpSoul

Your private AI operator platform — running entirely on your machine, no cloud account required to get started.

What is OpSoul?

OpSoul lets you create AI operators — purpose-built assistants with their own identity, knowledge, memory, and skills. Each operator is fully isolated and grows smarter over time through the GROW engine. Everything runs locally: your data never leaves your machine.

Creating Your First Operator

1
Open the Console — click the OpSoul icon in your menu bar (Mac) or system tray (Windows) and choose Open OpSoul Console, or navigate to http://localhost:3001 in any browser.
2
Register an account — on first launch you will see the Setup screen. Enter an email and password. This account is stored locally; it is the owner account for all operators.
3
Create an operator — go to Operators → New Operator. Give it a name and a short description of its purpose (e.g. "Customer support for Acme Inc.").
4
Add a soul — open the operator and go to the Soul tab. Write a personality description: tone, values, how it should respond. This is Layer 0 identity.
5
Seed knowledge — go to Knowledge Base and paste or upload documents the operator should know. Start with one or two entries and verify the operator uses them in chat.
6
Open a conversation — click Conversations → New and start chatting. The operator will use its soul, knowledge base, and memory to respond.

Connecting Your AI Model

OpSoul routes AI calls through OpenRouter, which gives you access to hundreds of models (Claude, GPT-4o, Gemini, Mistral, and more) with a single API key.

  1. Sign up at openrouter.ai and create an API key.
  2. In OpSoul, go to Settings → API Keys and paste your OpenRouter key.
  3. Choose a default model under Settings → Models. A good starting point is anthropic/claude-3-5-haiku (fast, affordable) or anthropic/claude-3-5-sonnet (more capable).
  4. Optionally set a per-operator model override in Operator → Settings.

The GROW Engine

GROW (Growth, Recall, Observation, Wisdom) is OpSoul's continuous-improvement layer. After each conversation the engine analyses what was discussed and proposes memory updates, knowledge additions, or soul refinements. You review and approve proposals before anything is committed.

The Three Lock Levels

Lock 0 Open — the operator can update its own memory and knowledge freely. Best for exploratory operators still being shaped.
Lock 1 Owner review — GROW proposes changes; you approve or reject each one before it takes effect. Recommended for production operators.
Lock 2 Frozen — no GROW proposals are generated. The operator's soul, knowledge, and memory are locked. Use for stable, certified operators.

Change the lock level per operator in Operator → Settings → GROW Lock.

Common Questions

How do I update OpSoul?

OpSoul checks for updates automatically when connected to the internet. When one is available, a notification appears in the tray menu. Click Install Update — the app restarts and your data is untouched.

Where is my data stored?

Everything lives in your OS user-data directory: ~/Library/Application Support/OpSoul/ on Mac or %APPDATA%\OpSoul\ on Windows. The embedded database files are in the pgdata/ subfolder there.

How do I back up my operators and data?

Quit OpSoul from the tray first, then copy the entire OpSoul/ user-data folder to a safe location. To restore, paste it back before relaunching. A built-in export/import feature is on the roadmap.

Can I run OpSoul on a server or share it with a team?

The desktop app is single-user. For team deployments, OpSoul can also run as a cloud service. Contact us at hello@opsoul.dev for hosted or self-hosted team plans.

The app shows "Startup failed" — what do I do?

Open Console.app (Mac) or Event Viewer (Windows) and search for "opsoul" to find detailed logs. Common causes: another process is using port 3001 or 5433. You can change the default ports in ~/.opsoul/config.json (create it if it doesn't exist) with "port": 3002.
